# Repository Guidance

## Tooling

- Use Vite+ (`vp`) as the command wrapper; do not run `pnpm`, `npx`, `vitest`, `oxlint`, or `wrangler` directly unless there is no `vp` route for the command.
- Install/update dependencies with `vp install`; add/remove packages with `vp add` / `vp remove` so the `pnpm@10.32.1` lockfile stays consistent.
- `vp dev`, `vp build`, and `vp test` are Vite+ built-ins, not `package.json` scripts. Run project scripts with `vp run <script>`.
- Current useful scripts: `vp run dev`, `vp run build`, `vp run preview`, `vp run deploy`, `vp run drizzle:generate`, `vp run drizzle:migrate`, `vp run drizzle:studio`, `vp run generate-types`.
- Validate with `vp check` and `vp test`. For focused server tests, pass the file path to `vp test`, e.g. `vp test server/notifiers/template.test.ts`.
- `vite.config.ts` configures staged hooks as `vp check --fix` and type-aware linting; `.vite-hooks/pre-commit` runs `vp staged`.
- Import Vite+/test helpers from `vite-plus` or `vite-plus/test`; do not add direct Vitest/Oxlint/Oxfmt dependencies.

## App Shape

- This is a Nuxt 4/Vue 3 uptime dashboard deployed as a Cloudflare Worker, not a plain Vite SPA. Nuxt entry is `app/app.vue`; server code lives under `server/`.
- Nitro uses `preset: "cloudflare-module"`; Wrangler deploys `.output/server/index.mjs` and serves `.output/public`.
- Scheduled checks are wired in both `nuxt.config.ts` (`nitro.scheduledTasks`) and `wrangler.toml` (`triggers.crons`) as `*/5 * * * *`; keep them aligned.
- Components are auto-imported from `app/components` with `pathPrefix: false`, so use names like `<AppButton>` rather than path-prefixed names.
- UI uses Element Plus. App theme/status tokens and small local view styles are in `app/assets/css/app.css`; prefer Element Plus components and the existing `app/components/ui/*` primitives.
- Dashboard copy is primarily Japanese; keep user-facing UI text consistent unless a feature already uses English.

## API And Data

- The executable API path is oRPC under `/rpc` (`server/routes/rpc/[...].ts` with `prefix: "/rpc"`). If README `/api/...` examples conflict, trust `server/contract.ts`, `server/router.ts`, and the client in `app/utils/rpc-client.ts`.
- Internal `/api` routes are incorrect for this project. Do not add, keep, or call `server/api/*` endpoints for app data; expose and consume all internal application APIs through oRPC instead.
- API contracts and OpenAPI metadata are declared in `server/contract.ts`; handlers are implemented in `server/router.ts`. Update both sides together.
- Browser calls should go through the typed oRPC client in `app/utils/rpc-client.ts`; it reads `auth_token` from `localStorage` and sends `Authorization: Bearer <token>`.
- Write operations compare the bearer token to `AUTH_PASSWORD`; local dev uses `.dev.vars` and production uses Cloudflare secrets.
- Drizzle schema is in `database/drizzle/schema/*`, query helpers in `database/drizzle/queries/*`, and generated migrations in `database/migrations`.
- After schema changes, run `vp run drizzle:generate`; apply local migrations with `vp exec wrangler d1 migrations apply healthcheck --local` and remote migrations with `vp run drizzle:migrate`.
- `worker-configuration.d.ts` is generated by `vp run generate-types`; `global.d.ts` intentionally widens generated `AUTH_PASSWORD` from the literal default to `string`.

## Tests And Verification

- Existing tests are Vitest-style files under `server/notifiers/*.test.ts` and import from `vite-plus/test`.
- Run `vp check` before handoff; run `vp test` for behavior changes, or a focused `vp test <file>` when only one notifier/test area changed.
- Do not commit `.dev.vars*`, `.wrangler/`, `.nuxt/`, `.output/`, or other generated/cache directories.

## Git

- When asked to commit, use a Japanese commit message with a type prefix such as `Feat:`, `Enhance:`, or `Fix:`.
